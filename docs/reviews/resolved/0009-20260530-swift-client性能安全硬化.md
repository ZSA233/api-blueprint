# Swift Client 性能安全硬化

- 状态：已修复
- 发现日期：2026-05-30
- 背景：Swift client 已具备 HTTP/SSE/WebSocket conformance，但从真实 iOS App 接入角度复查后，仍存在 URL force unwrap、byte stream 单字节 yield、JSON 成功路径 `Any` 中转、multipart header 注入和长连接 buffer 无界等性能/安全风险。
- 风险分级：P2
- 问题性质：生成运行时可用性、安全边界和性能风险。
- 存在性判断：确认存在。生成的 `URLSessionAPITransport` 曾包含 `URLComponents(...)!` / `components.url!`、`Data([byte])`、成功 JSON response 的 `JSONSerialization -> Any -> DTO` 路径、可变全局 `APICoding` scaffold，以及 multipart filename/header 未统一校验和转义的问题。
- 复现场景：无效 base URL 或 path 会 crash；大 byte stream 会产生大量小 `Data` 分配；恶意 multipart filename/header 可注入 CR/LF；SSE/WebSocket 单消息过大或消费慢时缺少明确限制；iOS app 需要注入自定义 JSON coding 时会落到可变全局状态。
- 影响范围：Swift client generated runtime、URLSession HTTP transport、STREAM/CHANNEL route surface、examples Swift conformance/Narrow package、benchmark 文档与可选 binary benchmark runner。
- 兼容性/修复风险：Swift generator 仍处于 pre-stable，本次接受破坏性更新。`APITransport.openStream/openChannel` 与生成的 stream/channel route 方法改为 `throws`；`APIRequest` JSON body 改为 typed encoder closure；`APICoding` preserved scaffold 推荐改为 factory 配置。
- 是否建议修复：是。该问题影响真实 iOS App 的协议 SDK 接入质量，且修复可以保持生成器边界，不引入 auth/retry/cache/session owner。
- 后续处置建议：继续把生产级登录、token refresh、retry、cache、前后台 session lifecycle 放在宿主 `APITransport` wrapper；若未来实现真正 typed Swift binary schema codec，再将 Swift binary benchmark 从 `Data` boundary 升级为字段级 packet 编解码基准。

## Resolution

- 修复日期：2026-05-30
- 修复摘要：
  - Swift runtime 新增 `APITransportError`、`APICodingConfig`，`HTTPAPIConfig` 增加 chunk/limit/buffer/coding/multipart knobs。
  - Swift URLSession transport 去除 URL force unwrap，JSON body/response 改用 typed encoder/decoder closure，byte stream 按 chunk 输出，error body/SSE/WebSocket payload 增加限制。
  - Multipart 增加 CR/LF 校验、quote/backslash 转义、非 ASCII `filename*=` 输出，并在 fileURL 或超过阈值时使用临时文件 request body stream。
  - STREAM/CHANNEL route 方法改为 `throws`，SSE/WebSocket bridge 使用有限 buffer 和 payload size 检查。
  - `scripts.example_benchmark binary` 增加 Swift target，使用临时 SwiftPM executable 依赖 `examples/swift` 的 `ABClientAPIRoutes` product。
- 验证命令：
  - `uv run api-gen generate -c examples/api-blueprint.toml --target swift.client`

## Binary Codec Follow-up

- 修复日期：2026-05-30
- 修复摘要：
  - Swift Markdown Binary Schema 不再以 `Data` 作为 packet payload 边界；route-local `GenBinary.swift` 生成字段级 packet struct、value constants、`Wire.encode/decode` 和兼容的 `encodeX/decodeX` helper。
  - Swift runtime binary 模块新增 `APIBinaryReader` / `APIBinaryWriter`、`APIBinaryEncodeError` / `APIBinaryDecodeError` 与字段路径包装 helper。
  - Swift codec 覆盖整数/浮点/bool/bytes/string、动态数组、nested struct、const、min/max、sizeof、assert、padding/reserved bytes 和 bitflags reserved mask 校验。
  - Swift conformance 与 binary benchmark 改为使用 typed packet 编解码，不再用任意 `Data` 透传。

## Binary Codec Wide Integer Follow-up

- 修复日期：2026-05-30
- 修复摘要：
  - Swift binary schema state now stores referenced integer fields using their generated Swift numeric type instead of forcing every state value through `Int`.
  - Dynamic byte/array counts convert referenced state through checked `apiBinaryCheckedInt` / `apiBinaryCheckedIntDecode` helpers, so oversized `u64` / `i64` values fail as structured `APIBinaryEncodeError` / `APIBinaryDecodeError` instead of reaching unsafe `Int(...)` conversion paths.
  - Numeric rule expressions, range checks, bitflag masks, and `u64` constants now render with type-aware `Int` / `Int64` / `UInt64` expressions.
  - Generated state now only includes fields that are actually referenced by counts or numeric rule expressions, reducing unnecessary mutable state in route-local binary codecs.
